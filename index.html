Rutas Para el Usuario

    Registro de Usuario

    Endpoint: POST /user/register

    Registrar un nuevo usuario en la aplicación.

    Parámetros de la solicitud

    • Email: (String) Dirección de correo electrónico del usuario.
    • Password: (String) Contraseña del usuario.
    • userName: (String) Nombre de usuario del usuario.

    Ejemplo de solicitud:

        
curl -X POST http://localhost:3000/user/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "usuario@example.com",
    "password": "contrasena123",
    "userName": "nombreusuario"
  }'

    Respuesta exitosa:

        
{
  "user": {
    "id": "123456789",
    "email": "usuario@example.com",
    "userName": "nombreusuario"
  },
  "token": "jwt-token-aqui"
}

    Respuesta de error:

        
{
  "error": "Usuario ya existe"
}

    Obtener Todos los Usuarios

    Endpoint: GET /user

    Obtiene la lista de todos los usuarios en la aplicación.

    Parámetros de la solicitud

    No hay parámetros requeridos.

    Ejemplo de solicitud:

        
curl http://localhost:3000/user

    Respuesta exitosa:

        
[
  {
    "id": "123456789",
    "email": "usuario1@example.com",
    "userName": "usuario1"
  },
  {
    "id": "987654321",
    "email": "usuario2@example.com",
    "userName": "usuario2"
  },
  // ...otros usuarios
]

    Obtener Usuario por ID

    Endpoint: GET /user/{userID}

    Obtiene la información de un usuario específico por su ID.

    Parámetros de la solicitud

    • {userID}: (String) ID único del usuario.

    Ejemplo de solicitud:

        
curl http://localhost:3000/user/123456789

    Respuesta exitosa:

        
{
  "id": "123456789",
  "email": "usuario1@example.com",
  "userName": "usuario1"
}

Rutas Para las Categorias (requieren Token del usuario tipo de token Jwt Bearer)


    Crear Nueva Categoría

    Endpoint: POST /category

    Crea una nueva categoría.

    Parámetros de la solicitud

    • name: (String) Nombre de la categoría.

    Ejemplo de solicitud:

        
curl -X POST http://localhost:3000/category \
  -H "Content-Type: application/json" \
  -d '{
    "name": "nueva_categoria"
  }'

    Respuesta exitosa:

        
{
  "_id": "123456789",
  "name": "nueva_categoria",
  "userId": "usuario_id"
}

    Respuesta de error:

        
{
  "error": "La categoría ya existe para este usuario."
}

    Obtener Todas las Categorías

    Endpoint: GET /category

    Obtiene la lista de todas las categorías del usuario.

    Parámetros de la solicitud

    No hay parámetros requeridos.

    Ejemplo de solicitud:

        
curl http://localhost:3000/category

    Respuesta exitosa:

        
[
  {
    "_id": "123456789",
    "name": "categoria1",

  },
  {
    "_id": "987654321",
    "name": "categoria2",

  },

Rutas para las Notas (requieren Token del usuario tipo de token Jwt Bearer)

    Obtener Todas las Notas

    Endpoint: GET /note

    Obtiene la lista de todas las notas del usuario.

    Parámetros de la solicitud

    No hay parámetros requeridos.

    Ejemplo de solicitud:

        
curl http://localhost:3000/note

    Respuesta exitosa:

        
[
  {
    "_id": "123456789",
    "title": "Nota 1",
    "content": "Contenido de la nota 1",
    "category": "ID de la categoría",
    "isActive": true
  },
  {
    "_id": "987654321",
    "title": "Nota 2",
    "content": "Contenido de la nota 2",
    "category": "ID de la categoría",
    "isActive": false
  },

]

    Agregar una Nueva Nota

    Endpoint: POST /note

    Agrega una nueva nota para el usuario.

    Parámetros de la solicitud

    • Title: (String) Título de la nota.
    • Content: (String) Contenido de la nota.
    • Category: (String) Nombre de la categoría.

    Ejemplo de solicitud:

        
curl -X POST http://localhost:3000/note \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Nueva Nota",
    "content": "Contenido de la nueva nota",
    "category": "Nombre de la categoría"
  }'

    Respuesta exitosa:

        
{
  "_id": "123456789",
  "title": "Nueva Nota",
  "content": "Contenido de la nueva nota",
  "category": "ID de la categoría",
  "isActive": true
}

    Respuesta de error:

        
{
  "error": "Error al agregar la nueva nota"
}

    Actualizar una Nota Existente

    Endpoint: PUT /note/{noteID}

    Actualiza una nota existente del usuario.

    Parámetros de la solicitud

    • Title: (String) Nuevo título de la nota.
    • Content: (String) Nuevo contenido de la nota.
    • IsActive: (Boolean) Nuevo estado de la nota (Activa/Inactiva).

    Ejemplo de solicitud:

        
curl -X PUT http://localhost:3000/note/123456789 \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Nuevo Título",
    "content": "Nuevo contenido",
    "isActive": false
  }'

    Respuesta exitosa:

        
{
  "_id": "123456789",
  "title": "Nuevo Título",
  "content": "Nuevo contenido",
  "category": "ID de la categoría",
  "isActive": false
}

    Respuesta de error:

        
{
  "error": "Error al actualizar la nota con ID 123456789"
}

    Eliminar una Nota

    Endpoint: DELETE /note/{noteID}

    Elimina una nota del usuario.

    Parámetros de la solicitud

    No hay parámetros requeridos.

    Ejemplo de solicitud:

        
curl -X DELETE http://localhost:3000/note/123456789

    Respuesta exitosa:

    Nota con ID 123456789 eliminada exitosamente.

    Respuesta de error:

        
{
  "error": "Error al eliminar la nota con ID 123456789"
}
