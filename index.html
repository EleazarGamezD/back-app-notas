Rutas Para el Usuario

    Documentación del Endpoint de Registro

    Registro de Usuario

    Endpoint: POST /user/register

    Registrar un nuevo usuario en la aplicación.

    Parámetros de la solicitud

    • Email: (String) Dirección de correo electrónico del usuario.
    • Password: (String) Contraseña del usuario.
    • userName: (String) Nombre de usuario del usuario.

    Ejemplo de solicitud:

        
curl -X POST http://localhost:3000/user/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "usuario@example.com",
    "password": "contrasena123",
    "userName": "nombreusuario"
  }'

    Respuesta exitosa:

        
{
  "user": {
    "id": "123456789",
    "email": "usuario@example.com",
    "userName": "nombreusuario"
  },
  "token": "jwt-token-aqui"
}

    Respuesta de error:

        
{
  "error": "Usuario ya existe"
}

    Inicio de Sesión

    Endpoint: POST /user/login

    Iniciar sesión en la aplicación.

    Parámetros de la solicitud

    • Email: (String) Dirección de correo electrónico del usuario.
    • Password: (String) Contraseña del usuario.

    Ejemplo de solicitud:

        
curl -X POST http://localhost:3000/user/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "usuario@example.com",
    "password": "contrasena123"
  }'

    Respuesta exitosa:

        
{
  "user": {
    "id": "123456789",
    "email": "usuario@example.com",
    "userName": "nombreusuario"
  },
  "token": "jwt-token-aqui"
}

    Respuesta de error:

        
{
  "error": "Usuario y/o contraseña invalida."
}

    Obtener Todos los Usuarios

    Endpoint: GET /user

    Obtiene la lista de todos los usuarios en la aplicación.

    Parámetros de la solicitud

    No hay parámetros requeridos.

    Ejemplo de solicitud:

        
curl http://localhost:3000/user

    Respuesta exitosa:

        
[
  {
    "id": "123456789",
    "email": "usuario1@example.com",
    "userName": "usuario1"
  },
  {
    "id": "987654321",
    "email": "usuario2@example.com",
    "userName": "usuario2"
  },
  // ...otros usuarios
]

    Obtener Usuario por ID

    Endpoint: GET /user/{userID}

    Obtiene la información de un usuario específico por su ID.

    Parámetros de la solicitud

    • {userID}: (String) ID único del usuario.

    Ejemplo de solicitud:

        
curl http://localhost:3000/user/123456789

    Respuesta exitosa:

        
{
  "id": "123456789",
  "email": "usuario1@example.com",
  "userName": "usuario1"
}
